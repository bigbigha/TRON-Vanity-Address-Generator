# TRON Vanity Address Generator

เครื่องมือสร้างที่อยู่ TRON แบบ vanity (กำหนดรูปแบบเองได้) ที่มีขนาดเล็กและเป็น **โอเพ่นซอร์ส**  
ออกแบบมาเพื่อให้สร้างที่อยู่ TRON ที่มีคำนำหน้า/คำต่อท้ายตามที่ต้องการได้อย่าง **ปลอดภัย** และ **ออฟไลน์ 100%**  
ใช้งานง่าย เร็ว และเหมาะทั้งผู้ใช้ทั่วไปและนักพัฒนา

## คุณสมบัติหลัก

- **ใช้งานแบบออฟไลน์ทั้งหมด**: ไม่มีการส่งคำขอออกอินเทอร์เน็ต ไม่มี telemetry ไม่มีการอัปโหลดข้อมูลอัตโนมัติ
- **โอเพ่นซอร์ส**: โค้ดมีขนาดเล็ก โฟกัสด้านเดียว ตรวจสอบและออดิทได้ง่าย
- **ทำงานด้วย CPU เท่านั้น**: ใช้ไบนารีเดียวแบบ static บน Linux, macOS และ Windows
- **รองรับหลายเธรด**: ใช้ทุกคอร์ของ CPU เพื่อเร่งความเร็วการค้นหา
- **การสุ่มที่ปลอดภัย**: ใช้ตัวสร้างเลขสุ่มที่ปลอดภัยในระดับเข้ารหัส (cryptographically secure RNG)

## สคริปต์ช่วยใช้งาน (ไม่ถนัดใช้คำสั่งก็ใช้ได้)

หากคุณไม่สะดวกพิมพ์คำสั่งใน Command Prompt / Terminal เอง  
คุณสามารถใช้สคริปต์ช่วยจากโฟลเดอร์ `scripts` ได้

- คำอธิบายแบบละเอียด: [`scripts/README.ru.md`](../scripts/README.ru.md)

### สรุป: สคริปต์สำหรับ Windows

1. ดาวน์โหลด `tronvanity-windows-amd64.exe` จากหน้า Releases
2. ดาวน์โหลด `windows-run.bat` จากโฟลเดอร์ `scripts`
3. นำทั้งสองไฟล์ไปไว้ในโฟลเดอร์เดียวกัน
4. ดับเบิลคลิก `windows-run.bat`
5. ทำตามคำแนะนำบนหน้าจอ และกรอก prefix / suffix / จำนวนที่อยู่ที่ต้องการให้ค้นหา

### สรุป: สคริปต์สำหรับ macOS / Linux

1. ดาวน์โหลดไบนารีตามระบบของคุณ:
    - `tronvanity-darwin-amd64` (Intel Mac)
    - `tronvanity-darwin-arm64` (Apple Silicon Mac)
    - `tronvanity-linux-amd64` (Linux)
2. ดาวน์โหลด `unix-run.sh` จากโฟลเดอร์ `scripts`
3. วางไบนารีและ `unix-run.sh` ไว้ในโฟลเดอร์เดียวกัน
4. เปิด Terminal, `cd` ไปยังโฟลเดอร์นั้น แล้วรัน `chmod +x unix-run.sh`
5. รัน `./unix-run.sh` และทำตามคำแนะนำบนหน้าจอ

สคริปต์เหล่านี้เป็นเพียงตัวหุ้ม (wrapper) ที่เรียกใช้ไบนารีในเครื่องของคุณเท่านั้น ไม่เชื่อมต่ออินเทอร์เน็ตและไม่แก้ไขไฟล์ระบบ

## เริ่มต้นใช้งานอย่างรวดเร็ว (ผู้ใช้ทั่วไป)

### Windows

1. ดาวน์โหลด `tronvanity-windows-amd64.exe`
2. วางไฟล์ในโฟลเดอร์ที่ต้องการ (หรือใช้ร่วมกับ `windows-run.bat` ก็ได้)
3. เปิด Command Prompt แล้ว `cd` ไปยังโฟลเดอร์นั้น
4. ตัวอย่างคำสั่ง:

```cmd
:: สร้างที่อยู่ที่ขึ้นต้นด้วย T888
tronvanity-windows-amd64.exe --prefix T888

:: สร้างที่อยู่ที่ลงท้ายด้วย 9999
tronvanity-windows-amd64.exe --suffix 9999
```

### macOS

1. ดาวน์โหลดไบนารีที่ตรงกับเครื่อง:
    - `tronvanity-darwin-amd64` (Intel)
    - `tronvanity-darwin-arm64` (Apple Silicon)
2. เปิด Terminal แล้ว `cd` ไปยังโฟลเดอร์ที่เก็บไฟล์
3. ให้สิทธิ์รันไฟล์ (ทำครั้งแรกครั้งเดียว):

```bash
chmod +x tronvanity-darwin-amd64      # Intel
chmod +x tronvanity-darwin-arm64      # Apple Silicon
```

4. ตัวอย่างคำสั่ง:

```bash
# สร้างที่อยู่ที่ขึ้นต้นด้วย T888
./tronvanity-darwin-amd64 --prefix T888

# สร้างที่อยู่ที่ลงท้ายด้วย 9999
./tronvanity-darwin-arm64 --suffix 9999
```

### Linux

1. ดาวน์โหลด `tronvanity-linux-amd64`
2. เปิด Terminal แล้ว `cd` ไปยังโฟลเดอร์ที่เก็บไฟล์
3. ให้สิทธิ์รัน:

```bash
chmod +x tronvanity-linux-amd64
```

4. ตัวอย่างคำสั่ง:

```bash
# สร้างที่อยู่ที่ขึ้นต้นด้วย T888
./tronvanity-linux-amd64 --prefix T888

# สร้างที่อยู่ที่ลงท้ายด้วย 9999
./tronvanity-linux-amd64 --suffix 9999
```

> แนะนำ: ถ้าจะใช้กับจำนวนเงินมาก ๆ ให้ย้ายโปรแกรมไปใช้งานบนเครื่องที่ออฟไลน์ (ไม่ต่ออินเทอร์เน็ต) เพื่อความปลอดภัยสูงสุด

## การใช้งานผ่านคำสั่ง (CLI)

ตัวอย่างพื้นฐาน:

```bash
# จับคู่เฉพาะคำนำหน้า (prefix)
tronvanity --prefix T888

# จับคู่เฉพาะคำต่อท้าย (suffix)
tronvanity --suffix 9999

# จับคู่ทั้ง prefix และ suffix ใช้ worker 8 ตัว และหยุดเมื่อเจอ 3 ที่อยู่
tronvanity --prefix T666 --suffix 888 --workers 8 --count 3

# สร้าง 5 ที่อยู่ที่ขึ้นต้นด้วย T123
tronvanity --prefix T123 --count 5
```

### พารามิเตอร์สำคัญ

- `--prefix` : คำนำหน้าที่ต้องการให้ที่อยู่ขึ้นต้น (เช่น `T888`)
- `--suffix` : คำต่อท้ายที่ต้องการให้ที่อยู่ลงท้าย (เช่น `9999`)
- `--workers`: จำนวน worker goroutine (ค่าปกติ = จำนวนคอร์ของ CPU)
- `--count`  : จำนวนที่อยู่ที่ต้องการหาให้ครบก่อนออกจากโปรแกรม (ค่าปกติ = 1)

## เคล็ดลับด้านความปลอดภัย

1. **สร้างคีย์แบบออฟไลน์**: หากจะใช้เก็บสินทรัพย์มูลค่าสูง แนะนำให้รันโปรแกรมบนเครื่องที่ไม่เชื่อมต่ออินเทอร์เน็ต
2. **ตรวจสอบที่อยู่ให้ดี**: ก่อนโอนเงินเข้าให้ตรวจสอบว่าที่อยู่ที่สร้างขึ้นมี prefix / suffix ตรงตามที่คุณตั้งใจ
3. **เก็บรักษา private key ให้ปลอดภัย**: ใช้ฮาร์ดแวร์วอลเล็ท ตัวจัดการรหัสผ่าน หรือแบ็กอัปแบบเข้ารหัสออฟไลน์ หลีกเลี่ยงการวาง private key ลงในเว็บหรือแอปที่ไม่น่าเชื่อถือ
4. **เริ่มจากจำนวนเงินน้อย ๆ**: ทดสอบการรับ–โอนด้วยจำนวนเล็ก ๆ ก่อน แล้วค่อยเพิ่มมูลค่าทีหลัง

## โปรแกรมทำงานอย่างไร (สรุปสั้น ๆ)

เครื่องมือนี้ใช้การ brute‑force ค้นหาที่อยู่ TRON vanity แบบโลคัลบนเครื่องของคุณ:

1. สร้าง private key ขนาด 32 ไบต์ด้วยตัวสร้างเลขสุ่มที่ปลอดภัยของ Go
2. สร้าง public key แบบไม่บีบอัด (65 ไบต์) บนเส้นโค้งวงรี secp256k1 (เหมือน TRON / Ethereum)
3. ตัดไบต์แรก `0x04` ออก แล้วทำ Keccak‑256 กับส่วนที่เหลือ จากนั้นดึง 20 ไบต์ท้าย
4. เติม prefix เครือข่ายหลักของ TRON คือ `0x41` ด้านหน้า → ได้ข้อมูลที่อยู่ 21 ไบต์
5. เข้ารหัสด้วย Base58Check → กลายเป็นที่อยู่ TRON รูปแบบมาตรฐาน `T...`
6. มี worker หลายตัวทำขั้นตอนนี้แบบขนาน และตรวจสอบว่าแต่ละที่อยู่ตรงกับ prefix / suffix ที่กำหนดหรือไม่ หากตรง ระบบจะพิมพ์ที่อยู่และ private key ออกทาง CLI

ทุกขั้นตอนทำงาน **บนเครื่องของคุณเท่านั้น** ไม่มีการเรียก API ภายนอก และไม่มีการส่ง private key ออกไปไหน
