# TRON Vanity Address Generator

Лёгкий, **open‑source** генератор TRON vanity‑адресов (кастомные шаблоны).  
**Безопасное**, **полностью офлайн** создание TRON‑адресов с нужным префиксом/суффиксом.  
Простой, быстрый и удобный для разработчиков.

## Возможности

- **Полностью офлайн**: нет сетевых запросов, телеметрии и авто‑загрузок.
- **Открытый исходный код**: небольшой и понятный код, который легко аудировать.
- **Только CPU**: один статический бинарник для Linux, macOS и Windows.
- **Многопоточность**: использует все доступные ядра CPU для ускорения перебора.
- **Криптографически безопасная случайность**: использует безопасный генератор случайных чисел.

## Вспомогательные скрипты (для тех, кто не любит консоль)

Если вы не хотите работать напрямую с командной строкой, можно использовать вспомогательные скрипты из папки `scripts`.

- Подробно: [`scripts/README.ru.md`](../scripts/README.ru.md)

### Кратко: скрипт для Windows

1. Скачайте `tronvanity-windows-amd64.exe` (релизы).
2. Скачайте `windows-run.bat` (папка `scripts`).
3. Поместите оба файла в одну папку.
4. Дважды щёлкните по `windows-run.bat`.
5. Следуйте подсказкам и укажите префикс, суффикс и количество адресов.

### Кратко: скрипт для macOS / Linux

1. Скачайте бинарник для вашей системы:
    - `tronvanity-darwin-amd64` (Intel Mac)
    - `tronvanity-darwin-arm64` (Apple Silicon Mac)
    - `tronvanity-linux-amd64` (Linux)
2. Скачайте `unix-run.sh` (папка `scripts`).
3. Положите бинарник и `unix-run.sh` в одну папку.
4. В терминале перейдите в эту папку, выполните `chmod +x unix-run.sh`.
5. Запустите `./unix-run.sh` и следуйте инструкциям.

Скрипты — это лишь оболочки вокруг локального бинарника; они не выходят в интернет и не изменяют системные файлы.

## Установка для разработчиков

```bash
# Клонировать репозиторий
git clone https://github.com/bigbigha/TRON-Vanity-Address-Generator.git
cd TRON-Vanity-Address-Generator

# Собрать бинарник
go build -o tronvanity ./cmd/tronvanity

# Пример запуска
./tronvanity --prefix T888
```

## Использование (CLI)

Базовые примеры:

```bash
# Только префикс
tronvanity --prefix T888

# Только суффикс
tronvanity --suffix 9999

# Префикс и суффикс, 8 рабочих потоков, выйти после 3 совпадений
tronvanity --prefix T666 --suffix 888 --workers 8 --count 3

# 5 адресов с префиксом T123
tronvanity --prefix T123 --count 5
```

### Аргументы командной строки

- `--prefix` — префикс для совпадения (например, `T888`).
- `--suffix` — суффикс для совпадения (например, `9999`).
- `--workers` — число worker‑горутин (по умолчанию: количество ядер CPU).
- `--count` — сколько совпадений найти перед выходом (по умолчанию: 1).

## Рекомендации по безопасности

1. **Генерируйте офлайн**. Для крупных сумм лучше использовать полностью офлайн‑машину.
2. **Проверяйте адреса**. Перед отправкой средств убедитесь, что адрес действительно содержит нужный префикс/суффикс.
3. **Храните приватные ключи безопасно**. Используйте аппаратные кошельки, менеджеры паролей или офлайн‑зашифрованные бэкапы. Никогда не вставляйте приватный ключ на сомнительные сайты или в неизвестные приложения.
4. **Начинайте с небольших сумм**. Перед тем как хранить большие объёмы средств, сделайте тестовые транзакции с небольшими суммами.

## Как это работает (кратко)

Инструмент выполняет локальный brute‑force поиск TRON vanity‑адресов:

1. Генерируется 32‑байтный приватный ключ с помощью криптографически безопасного генератора случайных чисел Go.
2. По нему считается несжатый публичный ключ (65 байт) на эллиптической кривой secp256k1 (как в TRON и Ethereum).
3. Отбрасывается первый байт `0x04`, к оставшимся байтам применяется Keccak‑256, берутся последние 20 байт.
4. Спереди добавляется префикс основной сети TRON `0x41` → получается 21 байт адресных данных.
5. Эти данные кодируются в Base58Check → стандартный TRON‑адрес вида `T...`.
6. Несколько worker‑потоков параллельно повторяют процесс и проверяют, соответствует ли адрес заданным условиям (префикс/суффикс). При совпадении адрес и приватный ключ выводятся в консоль.

Весь процесс выполняется **только на вашей машине**: сетевые запросы не выполняются, приватные ключи никуда не отправляются.
